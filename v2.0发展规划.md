### 阶段2：云服务器部署（1周）

#### 2.1 购买云服务器
```
推荐配置（阿里云ECS）：
- CPU: 2核
- 内存: 4GB
- 带宽: 5Mbps
- 系统盘: 40GB SSD
- 数据盘: 100GB
- 费用: 约200-300元/月

操作步骤：
1. 登录阿里云控制台
2. 购买ECS实例（选择Ubuntu 20.04）
3. 配置安全组（开放80、443、3000端口）
4. 绑定弹性公网IP
```

#### 2.2 数据库服务
```
选项1：阿里云RDS PostgreSQL（推荐）
- 规格: 1核2GB
- 存储: 20GB
- 费用: 约150元/月
- 优点: 自动备份、高可用

选项2：自建PostgreSQL
- 在ECS上安装PostgreSQL
- 费用: 0元（包含在ECS费用中）
- 需要自己维护和备份
```

#### 2.3 部署应用
```bash
# 1. 连接服务器
ssh root@your-server-ip

# 2. 安装必要软件
apt update
apt install -y nodejs npm postgresql nginx certbot

# 3. 克隆项目
git clone https://github.com/James-Fountain/MindLink.git
cd MindLink

# 4. 安装依赖
cd backend
npm install

# 5. 配置环境变量
cp .env.example .env
# 编辑.env文件，配置数据库等信息

# 6. 初始化数据库
psql -U postgres -d mindlink_db -f database/postgresql-schema.sql

# 7. 使用PM2管理进程
npm install -g pm2
pm2 start server.js --name mindlink-backend
pm2 save
pm2 startup

# 8. 配置Nginx反向代理
# 创建配置文件 /etc/nginx/sites-available/mindlink
```

#### 2.4 配置Nginx
```nginx
# /etc/nginx/sites-available/mindlink
server {
    listen 80;
    server_name mindlink.cn www.mindlink.cn;

    # 前端静态文件
    location / {
        root /var/www/mindlink;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

#### 2.5 配置HTTPS（SSL证书）
```bash
# 使用Let's Encrypt免费证书
certbot --nginx -d mindlink.cn -d www.mindlink.cn

# 自动续期
certbot renew --dry-run
```

```bash
# 数据库自动备份脚本
#!/bin/bash
# /root/backup-db.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backup/database"
DB_NAME="mindlink_db"

mkdir -p $BACKUP_DIR
pg_dump -U postgres $DB_NAME > $BACKUP_DIR/backup_$DATE.sql
gzip $BACKUP_DIR/backup_$DATE.sql

# 保留最近7天的备份
find $BACKUP_DIR -name "backup_*.sql.gz" -mtime +7 -delete

# 添加到crontab
# 0 2 * * * /root/backup-db.sh
```

```javascript
// frontend/js/features/scenario-guide.js
class ScenarioGuide {
  constructor() {
    this.currentScenario = null;
    this.currentStage = 0;
  }

  // 显示场景选择
  showScenarioSelection(userRole) {
    const scenarios = this.getScenariosForRole(userRole);

    // 渲染场景卡片
    scenarios.forEach(scenario => {
      this.renderScenarioCard({
        icon: scenario.icon,
        title: scenario.name,
        description: scenario.description,
        tags: scenario.tags
      });
    });
  }

  // 引导式对话
  startGuidedConversation(scenario) {
    this.currentScenario = scenario;
    this.currentStage = 0;

    // 显示当前阶段的引导问题
    this.showStageQuestions();
  }

  // 显示快捷回复
  showQuickReplies(stage) {
    const replies = this.currentScenario.quickReplies[stage];

    replies.forEach(reply => {
      this.renderQuickReplyButton(reply);
    });
  }
}
```

#### 3.3 数据库扩展
```sql
-- 场景表
CREATE TABLE scenarios (
    id VARCHAR(50) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    category VARCHAR(50),
    config JSONB,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 场景使用记录
CREATE TABLE scenario_usage (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    scenario_id VARCHAR(50) REFERENCES scenarios(id),
    conversation_id BIGINT REFERENCES conversations(id),
    stage VARCHAR(50),
    satisfaction_rating INTEGER,
    feedback TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 资源库
CREATE TABLE resources (
    id BIGSERIAL PRIMARY KEY,
    scenario_id VARCHAR(50) REFERENCES scenarios(id),
    type VARCHAR(50), -- 'template', 'guide', 'tool', 'article'
    title VARCHAR(200),
    content TEXT,
    file_url TEXT,
    tags TEXT[],
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
