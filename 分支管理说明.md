# 分支管理说明

## 📊 当前分支结构

```
main (v1.0.0)          ← 生产版本，已发布
  └── v1.0.0 tag       ← 版本标签

dev/v2.0               ← v2.0开发分支
  ├── v2.0规划文档
  ├── 云部署指南
  └── Git使用教程
```

## 🌿 分支说明

### main 分支
- **用途**: 生产环境，稳定版本
- **内容**: v1.0.0完整代码
- **保护**: 不直接在此分支开发
- **更新**: 只接受来自dev分支的合并

### dev/v2.0 分支
- **用途**: v2.0版本开发
- **内容**: v1.0.0代码 + v2.0规划文档
- **开发**: 所有v2.0的新功能都在此分支开发
- **合并**: 完成后合并回main并打v2.0.0标签

## 🔄 工作流程

### 1. 日常开发（在dev/v2.0分支）
```bash
# 切换到开发分支
git checkout dev/v2.0

# 拉取最新代码
git pull origin dev/v2.0

# 开发新功能...
# 修改文件

# 提交代码
git add .
git commit -m "feat: 添加新功能"

# 推送到远程
git push origin dev/v2.0
```

### 2. 创建功能分支（可选）
```bash
# 从dev/v2.0创建功能分支
git checkout dev/v2.0
git checkout -b feature/new-ui

# 开发功能...
git add .
git commit -m "feat: 新UI设计"

# 合并回dev/v2.0
git checkout dev/v2.0
git merge feature/new-ui

# 删除功能分支
git branch -d feature/new-ui
```

### 3. 发布v2.0版本
```bash
# 确保dev/v2.0分支完成所有开发
git checkout dev/v2.0

# 合并到main
git checkout main
git merge dev/v2.0

# 打标签
git tag -a v2.0.0 -m "v2.0.0发布"

# 推送
git push origin main
git push origin v2.0.0
```

## 🔍 查看分支差异

```bash
# 查看两个分支的差异
git diff main dev/v2.0

# 查看dev/v2.0新增的文件
git diff main dev/v2.0 --name-only

# 查看dev/v2.0新增的提交
git log main..dev/v2.0 --oneline
```

## 📝 提交规范

使用语义化提交信息：

```bash
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式
refactor: 重构
test: 测试
chore: 构建/工具

# 示例
git commit -m "feat: 添加考试焦虑辅导场景"
git commit -m "fix: 修复登录bug"

```
